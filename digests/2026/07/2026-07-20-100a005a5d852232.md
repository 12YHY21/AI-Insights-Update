**1. 小米机器人一号：基于超10万小时真实世界轨迹的视觉-语言-动作模型扩展**
原题：Xiaomi-Robotics-1: Scaling Vision-Language-Action Models with over 100K Hours of Real-World Trajectories
分类：多模态模型　推荐：⭐⭐⭐⭐（8.0/10）
来源：arXiv cs.RO　发布时间：2026-07-20
入选理由：大规模真实世界VLA模型，创新两阶段训练，性能领先，并提升关键基准。

**一句话结论**：小米提出Xiaomi-Robotics-1，一个在超10万小时UMI轨迹上预训练并通过跨具身后训练对齐机器人指令的基础VLA模型，在多项基准上达到最优性能。
**解决的问题**：机器人操作面临数据稀缺瓶颈，遥操作收集数据昂贵、缓慢且多样性受限，难以像LLM/VLM那样通过扩展数据量来提升策略的通用性和性能。

**核心技术**
- 使用超10万小时手持UMI设备收集的真实世界操作轨迹进行预训练，并利用自动标注流水线为轨迹片段生成状态转移自然语言描述，提供丰富语义监督。
- 采用两阶段训练：预训练赋予模型广域动作生成能力，后训练通过超过1万小时跨具身数据将能力对齐到机器人本体和人类常用的指令性任务提示。
- 模型架构基于Mixture-of-Transformers，结合预训练VLM（Qwen3-VL）和DiT，通过流匹配生成动作块，并在VLM上引入辅助Choice Policies加速收敛。
- 通过联合优化流匹配损失、回归损失和视觉语言下一token预测损失进行训练，并采用打包序列、复用VLM KV缓存等策略提高训练吞吐量。

**关键结果**
- 预训练阶段展现出有效的扩展规律：随数据量和模型尺寸增加，验证动作误差持续降低。
- 扩展效果直接传递至后训练：更强的预训练模型在未见环境的真实机器人评测中获得更高的开箱即用成功率。
- 在多个仿真基准中取得最优：RoboCasa365成功率57.6%（超越此前最优46.6%），RoboDojo平均分20.07（此前最优13.07）。
- 在复杂灵巧任务上微调数据效率高：平均每任务少于10小时数据，平均成功率75%，优于π0.5的40%。

**局限与风险**
- 原文未说明大规模训练的计算资源需求和能耗成本。
- 自动标注流水线可能产生噪声语言描述，对语义精确性的影响未评估。
- 真实世界开箱评测环境的具体数量和多样性未详细说明，泛化性边界不明确。

**适合阅读**：对机器人基础模型、视觉-语言-动作模型扩展以及多任务机器人学习感兴趣的研究人员和工程师。

**2. VarRate：无需训练的变速率KV缓存压缩，为长上下文LLM按需分配秩**
原题：VarRate: Training-Free Variable-Rate KV Cache Compression for Long-Context LLMs
分类：推理加速、量化与部署　推荐：⭐⭐⭐⭐（8.0/10）
来源：arXiv cs.CL　发布时间：2026-07-20
入选理由：免训练变秩KV缓存压缩，方法创新且效果显著，直接降低长上下文推理延迟。

**一句话结论**：VarRate是首个无需训练的低秩KV缓存编解码器，根据查询显著性为每个token分配可变秩，避免令牌丢弃和均匀分配，在查询无关重用下保持鲁棒，性能接近KVzip但压缩成本仅为其八分之一。
**解决的问题**：现有两大无训练KV缓存压缩方法各有缺陷：令牌选择方法（如SnapKV）在查询无关重用时重要性信号失效，且丢弃不可逆，导致准确率崩溃；均匀低秩编码为所有token分配相同秩，浪费预算。缺少一种无需训练、能根据重要性为每个token分配可变秩的方法。

**核心技术**
- 预计算共享低秩基：离线对少量校准数据收集每层的残差，通过SVD获得PCA基（每层一个R×D矩阵），无需训练。
- 显著性引导的秩分配：利用SnapKV的查询窗口注意力分数作为显著性，通过water-filling为每个token分配可变秩，确保每个token至少保留最小秩 r_min，不丢弃任何token。
- 残差编码与锚点保留：保留步长锚点和最近窗口token的精确表示，其余token编码为相对于最近锚点均值的残差，在预RoPE域操作。
- 固定预算下的秩再分配：在给定的总KV缓存预算内，将均秩编码器的预算重新分配为可变秩，保持总存储不变。

**关键结果**
- 在LongBench的16个任务上，使用20%的KV预算，VarRate在Llama-3.1-8B和Qwen2.5-7B上均保持与未压缩模型相差0.8个百分点以内，显著优于均秩消融实验。
- 在查询无关重用场景下，查询感知选择方法准确率下降11-15个百分点，而VarRate仅下降3.5-5.5个百分点，表现出强鲁棒性，且在所有预算下均压倒已发布的低秩编解码器Palu。
- VarRate的准确率与专门为查询无关重用设计的KVzip相当，在四分之三的设置中准确率等效，总体差异在一个百分点内，但压缩开销仅为KVzip的约八分之一。
- VarRate与量化正交，可与之组合使用，进一步降低内存占用。

**局限与风险**
- 未在所有基准上超越KVzip，KVzip在平均准确率上仍略有领先。
- 方法基于查询显著性信号，该信号在查询未知时可能不够理想（但通过保留最小秩缓解）。
- 当前仅针对预填充阶段压缩，解码阶段的动态分配未详细探讨。

**适合阅读**：关注长上下文LLM高效推理、KV缓存压缩的研究者和工程师，特别是寻求无需训练、鲁棒性强且内存高效方案的读者。
[查看原文](https://arxiv.org/abs/2607.15498)

**3. SkillCorpus：整合并评估开源技能生态以赋能真实世界LLM Agent**
原题：SkillCorpus: Consolidating and Evaluating the Open Skill Ecosystem for Real-World LLM Agents
分类：Agent 与工具调用　推荐：⭐⭐⭐⭐（8.0/10）
来源：arXiv cs.CL　发布时间：2026-07-20
入选理由：构建并评测技能语料库，直接提升Agent任务，系统性强且实用。

**一句话结论**：SkillCorpus 通过聚合、策展和匹配 821k 原始技能文件，构建了一个 96k 高质量技能语料库，并在三项真实基准上一致提升 Agent 性能，最高达 +7.5 pp。
**解决的问题**：开源 SKILL.md 技能生态碎片化、冗余且质量参差不齐，其在实际 Agent 任务中的价值不明确，缺乏整合、策展和系统评估。

**核心技术**
- 多源聚合与六阶段策展流水线：从 62 个源爬取 82.1 万原始文件，经过解析、去重、质量评分、安全过滤和许可筛选，产出 96,401 个高质量技能。
- 三维质量框架：从实用性、鲁棒性和安全性三个独立维度评估技能，并引入 19 个标志（包括 5 个硬门控）用于质量控制。
- 微调检索与选择栈：基于去重后的语料库训练双塔检索器，并结合 LLM 选择器，实现任务相关技能的精确匹配。
- 端到端真实基准评估：在 SkillsBench、GDPVal、QwenClawBench 三项基准上，使用两种 Agent 框架和多个开源到前沿骨干模型，量化技能语料库的实际增益。

**关键结果**
- 整合 SkillCorpus 在三项基准上一致提升性能，SkillsBench 上最大提升 +7.5 个百分点，其他基准也有显著增益。
- 增益受语料库覆盖度和 Agent 框架影响，覆盖度不足或框架限制会削弱效果。
- 安全性和许可过滤移除了约 4% 的技能，确保语料库可安全商用的 OSI 许可。
- 三维质量评分相互独立，安全性与其他维度弱相关，验证了分面评估的合理性。

**局限与风险**
- 质量评分依赖 LLM 评判，未进行人工标注验证，评分可靠性待进一步确认。
- 语料库偏向软件开发和数据处理领域，其他领域技能覆盖较少。
- 仅评估了文本到文本的 Agent 框架，未涉及多模态或代码执行环境。

**适合阅读**：对 LLM Agent 技能增强、开源资源策展和评估感兴趣的研究者和工程师。
